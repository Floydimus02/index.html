Host - Block the bullshit web [V3.0]

Advertisements, tracking scripts and other countless lines of code downloaded on to your machine to spam and monitor you whenever you connect to the internet.

This version leverages Oracle Cloud Infrastructure to host a Pi-Hole instance connected via OpenVPN, providing network-wide blocking for all your connected devices.

Note: This guide is written for Ubuntu OS (updated Dec 2025).

Phase 1: Oracle Cloud Infrastructure Setup

Initial setup of the virtual machine and network configuration within the Oracle Cloud console.

1.1 Create a Virtual Cloud Network (VCN)
  • Log in to your Oracle Cloud account.
  • Navigate to Networking ‚Üí Virtual Cloud Networks.
  • Click Start VCN Wizard.
  • Select VCN with Internet Connectivity and click Start VCN Wizard again.
  • Give your VCN a name (e.g., vpn-vcn). Click Next, then Create.
1.2 Create a Compute Instance
  • Navigate to Compute ‚Üí Instances and click Create Instance.
  • Name: vpn-server.
  • Image and shape: Click Edit.
    • Image: Select Canonical Ubuntu (latest LTS version).
    • Shape: Select Ampere series and the VM.Standard.A1.Flex shape.
  • Networking: Ensure your new VCN ( vpn-vcn) and the public subnet are selected.
  • Add SSH keys: Choose Generate a key pair for me and save the private key.
  • Click Create.
1.3 Configure Network Security Rules

Navigate to your VCN ‚Üí Security Lists ‚Üí Default Security List. Add the following Ingress Rules:

Source CIDR IP Protocol Port Range Description
0.0.0.0/0 UDP 1194 OpenVPN
0.0.0.0/0 TCP 80 Pi-Hole Web UI (HTTP)
0.0.0.0/0 TCP 443 Pi-Hole Web UI (HTTPS)
0.0.0.0/0 UDP 53 Pi-Hole DNS (UDP)
0.0.0.0/0 TCP 53 Pi-Hole DNS (TCP)

Phase 2: Software Installation & Configuration

Connect to your server and install the necessary software.

2.1 Connect & Elevate

Connect to your server using the public IP and the private key you downloaded.

chmod 400 /path/to/your/private-key.key ssh -i /path/to/your/private-key.key ubuntu@<YOUR_PUBLIC_IP>

Immediately after connecting, switch to the root user.

sudo su - root
2.2 System Update

Update all system packages.

apt update && apt upgrade -y
2.3 Configure OS Firewall (UFW)

Configure the server's local firewall to mirror the OCI security list.

apt install -y ufw ufw allow 22/tcp comment 'SSH Access' ufw allow 80/tcp comment 'Pi-Hole Web UI HTTP' ufw allow 443/tcp comment 'Pi-Hole Web UI HTTPS' ufw allow 1194/udp comment 'OpenVPN UDP' ufw allow 53/tcp comment 'Pi-Hole DNS TCP' ufw allow 53/udp comment 'Pi-Hole DNS UDP' ufw --force enable
2.4 Install Pi-Hole

Run the official Pi-Hole installation script.

curl -sSL https://install.pi-hole.net | bash

Installer Prompts Summary:

  • Upstream DNS: Choose any provider (e.g., Google).
  • Block Lists: Accept the default.
  • Protocols: Select both IPv4 and IPv6.
  • Accept defaults for the rest. Save the admin password shown at the end.

Set a new, strong admin password immediately.

pihole setpassword
2.5 Configure Pi-Hole for VPN

This setting is crucial for allowing VPN clients to use Pi-Hole as their DNS server.

  • Log in to the Pi-Hole web admin UI at http://<YOUR_PUBLIC_IP>/admin.
  • Go to Settings ‚Üí DNS tab.
  • Under Interface settings, select Permit all origins.
  • Click Save.
2.6 Configure Custom Blocklists

Manually add the following lists in the Pi-Hole web interface (Adlists page).

Lists to Enable

  • URL: https://raw.githubusercontent.com/StevenBlack/hosts/master/alternates/fakenews-gambling-porn-social/hosts
    Comment: Steven Black | Unified Host
  • URL: https://big.oisd.nl
    Comment: OISD | Big
  • URL: https://nsfw.oisd.nl
    Comment: OISD | Porn

List to Disable (For potential whitelisting)

  • URL: https://raw.githubusercontent.com/LukeSmithxyz/etc/master/ips
    Comment: Luke Smith | Master IPs

After adding all lists, navigate to Tools ‚Üí Update Gravity to apply the new blocklists.

Phase 3: OpenVPN Installation & Final Integration

Install OpenVPN and configure it to use Pi-Hole for DNS.

3.1 Install OpenVPN

Download and run the helper script on your server.

curl -O https://raw.githubusercontent.com/angristan/openvpn-install/master/openvpn-install.sh chmod +x openvpn-install.sh ./openvpn-install.sh

Crucial Installer Selections:

  • Protocol: Choose UDP.
  • DNS (CRITICAL): Select Custom DNS.
  • Custom DNS IP: Enter your server's Private IP Address (e.g., 10.0.0.X).
  • Client Name: Choose a name (e.g., my-phone).
3.2 Enable IP Forwarding

Edit /etc/sysctl.conf to enable IP forwarding permanently.

nano /etc/sysctl.conf

Uncomment the line #net.ipv4.ip_forward=1 by removing the #. Save and exit. Then apply:

sysctl -p
3.3 Move Client Config File

Move the generated .ovpn file to the ubuntu user's home directory for easy download.

# Replace <CLIENT_NAME> with the name you chose in step 3.1 mv /root/<CLIENT_NAME>.ovpn /home/ubuntu/<CLIENT_NAME>.ovpn
3.4 Download Client Config

On your local machine's terminal, run this command to download the file:

# Replace placeholders, the dot (.) downloads it to the current directory scp -i /path/to/your/private-key.key ubuntu@<YOUR_PUBLIC_IP>:/home/ubuntu/<CLIENT_NAME>.ovpn .
3.5 Test and Verify

Import the downloaded .ovpn file into your OpenVPN client (like OpenVPN Connect or Tunnelblick). Connect to the VPN and confirm that ads are being blocked by checking the Pi-Hole dashboard for query logs from your client's IP.

3.6 Auto-Connect on Ubuntu (Optional)

If using an Ubuntu desktop, run this command to manage network connections:

nm-connection-editor

Import your .ovpn file via the editor, then check the General tab and select 'Automatically connect to this network when it is available'.

Phase 4: Maintenance and Updates

Regularly update your server for security and performance.

4.1 Comprehensive System and App Update

Run this command as root to update the OS and Pi-Hole simultaneously.

apt update && apt upgrade -y && apt dist-upgrade -y && pihole -up
4.2 Update Pi-Hole Ad Lists (Gravity)

Run this to fetch the latest version of your blocklists, ensuring up-to-date ad protection.

pihole -g
4.3 Restart Server

Reboot the server after major updates or configuration changes.

reboot

Host - Block the bullshit web [V2.0]

Block advertisements, tracking scripts, and unwanted content across your devices by routing traffic through a Pi-Hole DNS resolver tunnelled via OpenVPN. This setup is hosted on the Oracle Cloud free tier.

Note: This guide is written for Ubuntu OS (updated Oct 2023).

Caution: We will enable root access for convenience. If you are familiar with SSH, standard security practices are recommended over this approach. Proceed with caution.

1. Instance Setup

  • Create an Oracle Cloud Free Tier account.
  • Setup an instance and switch the OS from Oracle Linux to Ubuntu.
  • Save the generated public and private keys carefully. Note your Public and Private IPs.
  • Log in to Shell and switch to root: sudo su - root.
  • Update your root password: passwd root.
  • Install Nano: sudo apt install nano -y.
  • Network Configuration: In the Oracle GUI, navigate to Networking > Virtual Cloud Networks > vcn-XXXX > Security Lists > Default Security List.
  • Add an Ingress Rule: Source CIDR 0.0.0.0/0, Protocol UDP, Destination Port 1194.

2. Software Installation

Connect to your server locally using your private key: ssh -i keyfile ubuntu@PublicIP.

  • Pi-Hole: Use the one-step automated install. Select interface tun0. Change password via sudo pihole -a -p.
  • Blocklists: I recommend the OISD blocklist for comprehensive coverage.
  • OpenVPN: Use the Angristan script. Select Port 1194.

3. Security & DNS Routing

  • Edit SSH config: sudo nano /etc/ssh/sshd_config.
  • Set PasswordAuthentication yes and PermitRootLogin yes (at your own risk).
  • Point VPN to Pi-Hole: sudo nano /etc/openvpn/server.conf and update to push "dhcp-option DNS [Pi-Hole Private IP]".
  • Restart SSH and the VM: sudo systemctl restart sshd.

4. Managing Profiles & Access

  • Profiles: Add/revoke certificates via ./openvpn-install.sh. Copy to home directory to download: cp /root/profile.ovpn /home.
  • Dashboard: Connect to the VPN, then navigate to https://[Private IP]/admin in your browser.
  • Maintenance: Periodically run apt update && apt upgrade, pihole -up, and pihole -g.

Host - Block the bullshit web [V1.0]

Advertisements, tracking scripts, and malicious code are constantly monitoring you. Host is a simple, effective way to block this noise‚Äîincluding trackers and pornographic content‚Äîusing a modified Windows hosts file.

This project was inspired by the "Bullshit Web" concept. I've also included a quick toggle feature specifically for Windows users.

Note: This guide is written for Windows 10 OS (updated Jun 2020).

1. Prerequisites & Resources

You will need two main components to get started:

  • Unified Hosts File: A consolidated list of reputable blocking sources with duplicates removed. Download here.
  • Toggle Script (hosts.bat): A batch script to enable/disable the hosts file with a single click. Download here.

2. The Setup

Follow these steps to configure your system for efficient blocking:

  • Installation: Place the downloaded hosts file in C:\Windows\System32\drivers\etc. Alternatively, you can fetch a specific variant from Steven Black‚Äôs GitHub repository.
  • Privilege Escalation: Navigate to HKEY_CLASSES_ROOT\batfile\shell in the Registry Editor. Change the (Default) value to runas to ensure batch files run with admin rights.
  • DNS Cache Opt-out: Go to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Dnscache and set the Start value to 4.
  • Service Check: Open services.msc and ensure the DNS Client is set to Disabled.

3. Efficient Usage

Manual editing is tedious. Use one of these methods to toggle the hosts file instantly:

Method 1: Direct Command
  • Save hosts.bat in C:\Windows\System32.
  • Trigger it anytime via Win + R -> type hosts -> Enter.
Method 2: Taskbar Shortcut
  • Pin Command Prompt to your taskbar.
  • Hold SHIFT, right-click the icon, and select Properties.
  • Change the Target to: %windir%\system32\cmd.exe /c "path\to\your\hosts.bat"
  • Rename it to Toggle Hosts for easy identification.
