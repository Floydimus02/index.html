                <h2>Host - Block the bullshit web [V3.0]</h2>

                <p>Advertisements, tracking scripts and other countless lines of code downloaded on to your machine to spam and monitor you whenever you connect to the internet.</p>

                <p>This version leverages Oracle Cloud Infrastructure to host a Pi-Hole instance connected via OpenVPN, providing network-wide blocking for all your connected devices.</p>

                

                <p><em>Note: This guide is written for Ubuntu OS (updated Dec 2025).</em></p>


                <h3>Phase 1: Oracle Cloud Infrastructure Setup</h3>

                <p>Initial setup of the virtual machine and network configuration within the Oracle Cloud console.</p>

                

                <strong>1.1 Create a Virtual Cloud Network (VCN)</strong>

                <ul class="setup-list">

                    <li>Log in to your Oracle Cloud account.</li>

                    <li>Navigate to <strong>Networking → Virtual Cloud Networks</strong>.</li>

                    <li>Click <strong>Start VCN Wizard</strong>.</li>

                    <li>Select <strong>VCN with Internet Connectivity</strong> and click Start VCN Wizard again.</li>

                    <li>Give your VCN a name (e.g., <span class="code-snippet">vpn-vcn</span>). Click Next, then Create.</li>

                </ul>


                <strong>1.2 Create a Compute Instance</strong>

                <ul class="setup-list">

                    <li>Navigate to <strong>Compute → Instances</strong> and click Create Instance.</li>

                    <li>Name: <span class="code-snippet">vpn-server</span>.</li>

                    <li>Image and shape: Click Edit.

                        <ul>

                            <li>Image: Select Canonical Ubuntu (latest LTS version).</li>

                            <li>Shape: Select Ampere series and the <span class="code-snippet">VM.Standard.A1.Flex</span> shape.</li>

                        </ul>

                    </li>

                    <li>Networking: Ensure your new VCN (<span class="code-snippet">vpn-vcn</span>) and the public subnet are selected.</li>

                    <li>Add SSH keys: Choose <strong>Generate a key pair for me</strong> and save the private key.</li>

                    <li>Click Create.</li>

                </ul>


                <strong>1.3 Configure Network Security Rules</strong>

                <p>Navigate to your VCN → Security Lists → Default Security List. Add the following Ingress Rules:</p>

                    <thead>

                        <tr>

                        <tr><td>0.0.0.0/0</td><td>TCP</td><td>80</td><td>Pi-Hole Web UI (HTTP)</td></tr>

                        <tr><td>0.0.0.0/0</td><td>TCP</td><td>443</td><td>Pi-Hole Web UI (HTTPS)</td></tr>

                        <tr><td>0.0.0.0/0</td><td>UDP</td><td>53</td><td>Pi-Hole DNS (UDP)</td></tr>

                <h3>Phase 2: Software Installation & Configuration</h3>

                <p>Connect to your server and install the necessary software.</p>

                

                <strong>2.1 Connect & Elevate</strong>

                <p>Connect to your server using the public IP and the private key you downloaded.</p>

                <div class="code-block">chmod 400 /path/to/your/private-key.key

ssh -i /path/to/your/private-key.key ubuntu@&lt;YOUR_PUBLIC_IP&gt;</div>

                <p>Immediately after connecting, switch to the root user.</p>

                <div class="code-block">sudo su - root</div>


                <strong>2.2 System Update</strong>

                <p>Update all system packages.</p>

                <div class="code-block">apt update && apt upgrade -y</div>


                <strong>2.3 Configure OS Firewall (UFW)</strong>

                <p>Configure the server's local firewall to mirror the OCI security list.</p>

                <div class="code-block">apt install -y ufw

ufw allow 22/tcp comment 'SSH Access'

ufw allow 80/tcp comment 'Pi-Hole Web UI HTTP'

ufw allow 443/tcp comment 'Pi-Hole Web UI HTTPS'

ufw allow 1194/udp comment 'OpenVPN UDP'

ufw allow 53/tcp comment 'Pi-Hole DNS TCP'

ufw allow 53/udp comment 'Pi-Hole DNS UDP'

ufw --force enable</div>


                <strong>2.4 Install Pi-Hole</strong>

                <p>Run the official Pi-Hole installation script.</p>

                <div class="code-block">curl -sSL https://install.pi-hole.net | bash</div>

                

                <p><strong>Installer Prompts Summary:</strong></p>

                <ul class="setup-list">

                    <li>Upstream DNS: Choose any provider (e.g., Google).</li>

                    <li>Block Lists: Accept the default.</li>

                    <li>Protocols: Select both IPv4 and IPv6.</li>

                    <li>Accept defaults for the rest. Save the admin password shown at the end.</li>

                </ul>


                <p>Set a new, strong admin password immediately.</p>

                <div class="code-block">pihole setpassword</div>


                <strong>2.5 Configure Pi-Hole for VPN</strong>

                <p>This setting is crucial for allowing VPN clients to use Pi-Hole as their DNS server.</p>

                <ul class="setup-list">

                    <li>Log in to the Pi-Hole web admin UI at <span class="code-snippet">http://&lt;YOUR_PUBLIC_IP&gt;/admin</span>.</li>

                    <li>Go to <strong>Settings → DNS</strong> tab.</li>

                    <li>Under Interface settings, select <strong>Permit all origins</strong>.</li>

                    <li>Click Save.</li>

                </ul>


                <strong>2.6 Configure Custom Blocklists</strong>

                <p>Manually add the following lists in the Pi-Hole web interface (Adlists page).</p>

                

                <p><strong>Lists to Enable</strong></p>

                <ul class="setup-list">

                    <li>URL: <span class="code-snippet">https://raw.githubusercontent.com/StevenBlack/hosts/master/alternates/fakenews-gambling-porn-social/hosts</span><br>Comment: Steven Black | Unified Host</li>

                    <li>URL: <span class="code-snippet">https://big.oisd.nl</span><br>Comment: OISD | Big</li>

                    <li>URL: <span class="code-snippet">https://nsfw.oisd.nl</span><br>Comment: OISD | Porn</li>

                </ul>


                <p><strong>List to Disable (For potential whitelisting)</strong></p>

                <ul class="setup-list">

                    <li>URL: <span class="code-snippet">https://raw.githubusercontent.com/LukeSmithxyz/etc/master/ips</span><br>Comment: Luke Smith | Master IPs</li>

                </ul>


                <p>After adding all lists, navigate to <strong>Tools → Update Gravity</strong> to apply the new blocklists.</p>


                <h3>Phase 3: OpenVPN Installation & Final Integration</h3>

                <p>Install OpenVPN and configure it to use Pi-Hole for DNS.</p>

                

                <strong>3.1 Install OpenVPN</strong>

                <p>Download and run the helper script on your server.</p>

                <div class="code-block">curl -O https://raw.githubusercontent.com/angristan/openvpn-install/master/openvpn-install.sh

chmod +x openvpn-install.sh

./openvpn-install.sh</div>

                

                <p><strong>Crucial Installer Selections:</strong></p>

                <ul class="setup-list">

                    <li>Protocol: Choose <strong>UDP</strong>.</li>

                    <li>DNS (CRITICAL): Select <strong>Custom DNS</strong>.</li>

                    <li>Custom DNS IP: Enter your server's <strong>Private IP Address</strong> (e.g., <span class="code-snippet">10.0.0.X</span>).</li>

                    <li>Client Name: Choose a name (e.g., <span class="code-snippet">my-phone</span>).</li>

                </ul>


                <strong>3.2 Enable IP Forwarding</strong>

                <p>Edit <span class="code-snippet">/etc/sysctl.conf</span> to enable IP forwarding permanently.</p>

                <div class="code-block">nano /etc/sysctl.conf</div>

                <p>Uncomment the line <span class="code-snippet">#net.ipv4.ip_forward=1</span> by removing the <span class="code-snippet">#</span>. Save and exit. Then apply:</p>

                <div class="code-block">sysctl -p</div>


                <strong>3.3 Move Client Config File</strong>

                <p>Move the generated <span class="code-snippet">.ovpn</span> file to the <span class="code-snippet">ubuntu</span> user's home directory for easy download.</p>

                <div class="code-block"># Replace &lt;CLIENT_NAME&gt; with the name you chose in step 3.1

mv /root/&lt;CLIENT_NAME&gt;.ovpn /home/ubuntu/&lt;CLIENT_NAME&gt;.ovpn</div>


                <strong>3.4 Download Client Config</strong>

                <p>On your local machine's terminal, run this command to download the file:</p>

                <div class="code-block"># Replace placeholders, the dot (.) downloads it to the current directory

scp -i /path/to/your/private-key.key ubuntu@&lt;YOUR_PUBLIC_IP&gt;:/home/ubuntu/&lt;CLIENT_NAME&gt;.ovpn .</div>


                <strong>3.5 Test and Verify</strong>

                <p>Import the downloaded <span class="code-snippet">.ovpn</span> file into your OpenVPN client (like OpenVPN Connect or Tunnelblick). Connect to the VPN and confirm that ads are being blocked by checking the Pi-Hole dashboard for query logs from your client's IP.</p>


                <strong>3.6 Auto-Connect on Ubuntu (Optional)</strong>

                <p>If using an Ubuntu desktop, run this command to manage network connections:</p>

                <div class="code-block">nm-connection-editor</div>

                <p>Import your <span class="code-snippet">.ovpn</span> file via the editor, then check the General tab and select 'Automatically connect to this network when it is available'.</p>


                <h3>Phase 4: Maintenance and Updates</h3>

                <p>Regularly update your server for security and performance.</p>


                <strong>4.1 Comprehensive System and App Update</strong>

                <p>Run this command as root to update the OS and Pi-Hole simultaneously.</p>

                <div class="code-block">apt update && apt upgrade -y && apt dist-upgrade -y && pihole -up</div>


                <strong>4.2 Update Pi-Hole Ad Lists (Gravity)</strong>

                <p>Run this to fetch the latest version of your blocklists, ensuring up-to-date ad protection.</p>

                <div class="code-block">pihole -g</div>


                <strong>4.3 Restart Server</strong>

                <p>Reboot the server after major updates or configuration changes.</p>

                <div class="code-block">reboot</div>

                <div class="caution-box">

                <h3>1. Instance Setup</h3>

                <ul class="setup-list">

                    <li>Setup an instance and switch the OS from Oracle Linux to <strong>Ubuntu</strong>.</li>

                    <li>Save the generated public and private keys carefully. Note your Public and Private IPs.</li>

                    <li>Log in to Shell and switch to root: <span class="code-snippet">sudo su - root</span>.</li>

                    <li>Update your root password: <span class="code-snippet">passwd root</span>.</li>

                    <li>Install Nano: <span class="code-snippet">sudo apt install nano -y</span>.</li>

                    <li><strong>Network Configuration:</strong> In the Oracle GUI, navigate to <em>Networking > Virtual Cloud Networks > vcn-XXXX > Security Lists > Default Security List</em>.</li>

                    <li>Add an <strong>Ingress Rule</strong>: Source CIDR <span class="code-snippet">0.0.0.0/0</span>, Protocol <strong>UDP</strong>, Destination Port <strong>1194</strong>.</li>

                </ul>

                <p>Connect to your server locally using your private key: <span class="code-snippet">ssh -i keyfile ubuntu@PublicIP</span>.</p>

                <ul class="setup-list">

Select interface <span class="code-snippet">tun0</span>. Change password via <span class="code-snippet">sudo pihole -a -p</span>.</li>

                    <li><strong>Blocklists:</strong> I recommend the <a href="https://oisd.nl/" target="_blank">OISD blocklist</a> for comprehensive coverage.</li>

                    <li><strong>OpenVPN:</strong> Use the <a href="https://github.com/angristan/openvpn-install" target="_blank">Angristan script</a>. Select Port <strong>1194</strong>.</li>

                </ul>


                <h3>3. Security & DNS Routing</h3>

                <ul class="setup-list">

                    <li>Edit SSH config: <span class="code-snippet">sudo nano /etc/ssh/sshd_config</span>.</li>

                    <li>Set <span class="code-snippet">PasswordAuthentication yes</span> and <span class="code-snippet">PermitRootLogin yes</span> (at your own risk).</li>

                    <li>Point VPN to Pi-Hole: <span class="code-snippet">sudo nano /etc/openvpn/server.conf</span> and update to <span class="code-snippet">push "dhcp-option DNS [Pi-Hole Private IP]"</span>.</li>

                    <li>Restart SSH and the VM: <span class="code-snippet">sudo systemctl restart sshd</span>.</li>

                </ul>


                <h3>4. Managing Profiles & Access</h3>

                <ul class="setup-list">

                    <li><strong>Dashboard:</strong> Connect to the VPN, then navigate to <span class="code-snippet">https://[Private IP]/admin</span> in your browser.</li>

                    <li><strong>Maintenance:</strong> Periodically run <span class="code-snippet">apt update && apt upgrade</span>, <span class="code-snippet">pihole -up</span>, and <span class="code-snippet">pihole -g</span>.</li>

        function showTab(tabId) {

            const contents = document.querySelectorAll('.tab-content');

            contents.forEach(content => content.classList.remove('active'));


            const navLinks = document.querySelectorAll('#main-nav a');

            navLinks.forEach(link => link.classList.remove('active'));


            // Show the target section

            const targetSection = document.getElementById(tabId);

            if (targetSection) {

                targetSection.classList.add('active');

            }


            // Highlight the nav link ONLY if it's one of the main header tabs

            const navLink = document.getElementById('nav-' + tabId);

            if (navLink) {

                navLink.classList.add('active');

            } else if (tabId.startsWith('host-project') || tabId === 'rhythm-stats') {

                // Keep Side Quests highlighted if in a project sub-page

                document.getElementById('nav-side-quests').classList.add('active');

            }


            window.scrollTo(0, 0);

        }
